## 测试原则
1. **边界优先**：优先测试边界情况、错误处理、异常输入
2. **不修改源码**：所有测试仅验证逻辑，不修改现有代码
3. **最小依赖**：尽量使用 mock 避免外部依赖
4. **覆盖率为辅**：重点关注逻辑覆盖，而非单纯的行覆盖率
5. **可复现**：所有测试用例可重复执行，结果稳定
